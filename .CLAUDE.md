

- Minimize COSMETIC CHANGES (e.g., changing a parameter name of a method or a comment unnecessarily)
- Before making any change, understand the full context by reading as much code as necessary. Do not guess what you can know for certain by reading code.

- No hacky workarounds. No slop or code bloat.
- Do not just focus on fixing the symptoms of the problem. Fix the root cause.
- DO NOT OVERCOMPLICATE THE CODE.

- List all the issues in the log messages and work on the root cause of each one. Reduce the guesswork. Find all instances of each issue in the code.

- Fail the code if it violates expectations. Data needs to be typed correctly. Objects need to have the expected attributes and methods. If something is not as expected, raise an error immediately with a clear message. Do not let bad data propagate through the system.
